Проект реализован на .NET 10, БД - MSSQL через EF. 
Поскольку доменная область простая, использована N-layer архитектура. При разрастании доменной логики BL можно легко трансформировать в Application и выделить доменную логику в Domain.

Сторонние библиотеки:
* Для валидации - FluentValidation
* Swagger
* Для маппинга был использован Mappster вместо AutoMaper скорее в тестовых целях. По результатам - работает нормально, конфигурация несложная.
* Для склейки ExpressionTree использовалась готовая библиотека LinqKit. 
* Для парсинга диапазонов дат использовался Hl7.Fhir, тесты на парсинг - xUnit.

Примечания:
* UOW/AppContext в Bl - поскольку доменная область в требованиях очень маленькая, UOW пока что не выделял. 
* ErrorHandlingMiddleware - заглушка для подключения логирования в будущем.

Структура репозитория:
* TestTask - сам сервис
* DataSeedApp - консольное приложение для заполнения данными
* PostmanConfiguration - .json с готовой коллекцией для тестирования
* scripts - ps1 скрипты для запуска/обновления контейнеров, сетапа локального окружения
* .env - убрана из гит игнора исключительно в тестовых целях, что бы не нужно было создавать для локального запуска
